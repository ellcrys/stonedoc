Ownership Block
This is an optional block containing information about the ownership of a stone instance. It is reserved for the host to store and determine ownership of an asset by associating the stone with an address known to it.

Only the address owner who has access to the address's private key can authorize transactions against a stone belonging to their address.

A stone without ownership information will be consider to be an unowned asset by a host. Such an asset can be freely transferred by anyone to any address within the host.
Structure of an ownership block
			
{
   "ref_id":  "jh88hg3sxn43isjhdg0jsdjie03cu3",
   "type":    "sole",
   "sole": {
      "address_id": "jf8asdjji904fcn09jtunc909du3nec0envu9rg9"
   }
}
ref_id (type: string, required: yes): This is the id of the stone where the ownership block is included. If the value is not equal to meta.id, the block is considered invalid.

type (type: string, required: yes): This is the type of ownership. Only sole ownership is currently supported. Expected value should be sole.

sole (type: object, required: no): Optional. This object contains information about the address that owns the stone. This is required when ownership.type is set to sole.

sole.address_id (type: string, required: yes): This is the address id associated with the stone instance. Only signatures signed with the private key of this address will be accepted by the host.

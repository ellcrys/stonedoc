Blocks
A stone is represented in JSON format. It is made up of five properties know as "blocks". The names of these blocks are meta, ownership, attributes, embeds and signatures.
Structure of a stone object
			
{
   "meta":       	{},
   "ownership":  	{},
   "attributes": 	{},
   "embeds":	 	{},
   "signatures": 	{}
}
Meta: The meta block is where the header information about the stone and it's issuer is stored. This block must be signed by the issuer during creation.

Ownership: An optional block for describing ownership. This block is used by the host to determine and describe ownership. It must be signed by the host.

Attributes: This is the only place an issuer can store arbitrary data about a stone instance. It must be signed by the issuer.

Embeds: To support complex stones, this block allows the issuer to include one or more stones inside of another stone. This is suitable for assets that also come with additional assets. This block must be signed by the issuer.

Signatures: This is where the JWS signatures of the other blocks is stored. It is also the block that gets base64 encoded to make up the final stone token.

Each block accepts an object containing expected properties. For a stone to be considered valid, all blocks must pass validation checks. The signatures block is excluded from the signing and verification processes.

Structure of a signatures block
			
{
   "meta":       	"xxxxxxxx",
   "ownership":  	"xxxxxxxx",
   "attributes": 	"xxxxxxxx",
   "embeds":	 	"xxxxxxxx"
}

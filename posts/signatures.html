Signatures Block
The signatures block contains the JWS signature/token of all the other blocks. The final stone token is the base64 encoded string of the signature block.

The benefit of using JWS for signing is the conveniece of rebuilding the full stone's object from the encoded payload in the token.

A stone will be considered invalid if a block's signature could not be verified using the issuer or host private keys.
Structure of a signatures block
			
{
   "meta": "eyk829xnd...hga0xa3aop1wsk",
   "ownership": "eqhauukfnh...ahhej23ndaa93smad",
   "attributes": "ey28ehdba9fhj...929uskhf8",
   "embeds": "8dhhdy289duh...2wjxjnjeks"
}

Signatures Block
The signatures block contains the JWS signatures of all the other blocks. The base64 encoded string of this block is what gets transferred between users and applications. The benefit of using JWS for signing is the ability to rebuild a stone's JSON object from the signature. A stone will be considered invalid if a block's signature could not be verified.
Structure of a signatures block
			
{
   "meta": "eyk829xnd...hga0xa3aop1wsk",
   "ownership": "eqhauukfnh...ahhej23ndaa93smad",
   "attributes": "ey28ehdba9fhj...929uskhf8",
   "embeds": "8dhhdy289duh...2wjxjnjeks"
}
